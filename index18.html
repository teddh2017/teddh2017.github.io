1. HTTP프로토콜이란?

→ Hyper Text Transfer Protocol 의 줄임말로써 인터넷 상에서 pc간 하이퍼텍스트 문서의 교환(통신)을 위한 프로토콜입니다. 그리고 모든 통신에는 상방간의 미리 규정된 통신 규약이 있습니다. HTTP 역시 이런 통신규약의 일종이며 웹 서버/클라이언트에서는 이 규약을 이해하고 처리할 수 있어야 한다. 또한 웹 서버에서는 HTTP 데몬이, 클라이언트는 웹 브라우저가 그 역할을 담당합니다.
정확히 말하면 웹 서버의 HTTP 데몬과 웹 브라우저의 통신을 위한 통신 규약을 말하는 것이다.
HTTP 프로토콜을 이용해서 웹상에서 파일(텍스트, 이미지, 사운드, 비디오 등)을 송, 수신 할 수 있다. 마지막으로 HTTP의 통신 역시 TCP 통신상에서 이루어진다는 점을 기억해야 한다.

◆ 위에서 말한 것 중에 더하여 애기하자면 여기서 하이퍼 텍스트란?

→ 관련된 문서를 하이퍼링크로 연결하여 검색할 수 있도록 만들어진 문서형식입니다.

2. 메소드 get, post 전송방식을 조사하시오.

1)GET 방식

→ GET 방식은 URL을 통해 변수를 전달하는 것이다. GET 방식은 서버에 있는 정보를 가져오기 위해 설계된 방법이며 할 수 있는 데이터 크기는 최대 240바이트이다.
또한 전송할 데이터를 문자열 형태로 URL의 뒤에 인수로 붙여서 전송합니다.
반면 Form 태그에서는 'GET'방식을 서버로 전송 시 서버의 처리가 지연 되었을 경우 중복해서 요청을 하게 되는 문제가 발생함으로 사용을 하지 않습니다.
더하여 GET 방식은 POST 방식과 같이 폼 태그를 사용해서 전송도 하고 링크에 직접 사용해도 되고 주소창에 직접 입력해도 전송 된다.
반면에 보안성이 없어 누구나 전송 내용을 볼 수 있다는 단점이 있습니다.

2)POST 방식

→ POST 방식은 URL 헤더가 아닌 패킷의 Body에 변수에 담아서 넘겨주는 것이다. POST 방식은 서버로 정보를 올리기 위해 설계된 방법이며 데이터 크기에 제한이 없다.
또한 URL 파라미터 표시가 안 된다. 더하여 파일의 형태로 전송됨으로 URL상에 나타나지 않아서 보안성이 있다.
예를 들어 id나 패스워드는 POST 방식으로 사용 하는 것이 좋다.
그리고 POST 방식은 정보가 숨겨져서 데이터를 전송하기 때문에 눈으로는 볼 수가 없다.
더하여 GET방식에 비해서 대용량의 데이터를 전송할 수 있습니다.

→ 그리고 위에서 본 것과 같이 그냥 정리해보면 GET 방식은 용량이 작은 경우에 많이 쓰고 POST 방식은 용량이 큰 경우 쓴다는 점을 기억하자.
